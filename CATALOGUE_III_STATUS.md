## Challenges & Recommendations

### 1. **File Size Management**
- Special Universal Connectors category is HUGE (200+ products)
- **Recommendation**: Split into multiple files by pole count
  - `special-connectors-1-2-pole.ts`
  - `special-connectors-3-4-pole.ts`
  - `special-connectors-5-6-pole.ts`
  - `special-connectors-ecm-high-pole.ts`

### 2. **Product Complexity**
- Many products have similar names with slight variations
- Vehicle-specific variants (Tata, Ashok Leyland, Mahindra, Maruti)
- **Recommendation**: Use detailed slugs and tags for differentiation

### 3. **Part Number Format**
- Consistent format: SH + 7 digits + F/M suffix for gender
- ECM connectors use longer part numbers
- Some have "OLD-" prefix indicating superseded parts

### 4. **Wire Specifications**
- Standard: 9" long deluxe wire
- ECM: 15" long wire
- Various wire gauges: 4mm, 5mm, 6mm, 8mm
- Special wire vs Deluxe wire

### 5. **Configuration Strategy**
Given the massive size, I recommend:
- **Phase 1**: Complete universal-holders.ts (add remaining ~25 products)
- **Phase 2**: Create smaller connector category files (flasher, headlamp, JPT, sensor, fuse)
- **Phase 3**: Create special-universal-connectors split files (200+ products)
- **Phase 4**: Create relay-with-base and jointer files
- **Phase 5**: Update products.config.ts to aggregate all new categories
